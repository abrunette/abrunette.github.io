ローマ字 → English GitHub LinkCompleted: 2017/12/17
A Romaji to English translator.

This project required us to design and implement a scanner, parser, and translator for Japanese romaji into english.

We implemented it in three parts, scanner, parser, and translator, building on the previous parts to add more functionality. The scanner scans words in a text file and determines if they are valid Japanese words. That is to say, they match up with the Japanese syllabary using a DFA map. If they do, they are matched to a certain type to be used later in the parser. Otherwise they are marked ERROR.

In the parser, the words are checked, in order, after they are scanned to determine if they match a pre-defined BNF grammar rule. If they reach the end, then the sentence is valid, otherwise it is wrong and the user has an option to correct it or end the program.

Within the parser functions there are calls to translation functions. If the words are matched to a valid english word they will be replaced in the output.

Several arrays and a map are used to match words to their respective types or translations. There are also some extra functions to improve quality of life for the user, allow for debugging, and make the output easier to understand.


Virtual Memory Manager GitHub LinkCompleted: 2017/10/25
A program that uses a TLB and paging table to manage physical and virtual memory.

This project consisted of writing a program that translates logical to physical addresses for a virtual address space of size 216 = 65,536 bytes. The program will read from a file containing logical addresses and, using a TLB as well as a page table, will translate each logical address to its corresponding physical address and output the value of the byte stored at the translated physical address. The goal behind this project is to simulate the steps involved in translating logical to physical addresses.

The program will read a file containing several 32-bit integer numbers that represent logical addresses. However, it only looks at 16-bit addresses, so it masks the rightmost 16 bits of each logical address. These 16 bits are divided into an 8-bit page number and an 8-bit page offset.

Memory specifics:

  • 28 entries in the page table
  • Page size of 28 bytes
  • 16 entries in the TLB
  • Frame size of 28 bytes
  • 256 frames
  • Physical memory of 65,536 bytes (256 frames × 256-byte frame size)

Fibonacci Search Algorithm GitHub LinkCompleted: 2017/03/14
Searches through an array using the fibonacci sequence as the parameter. An extra credit assignment exploring alternative searching algorithms.

This algorithm uses fibonacci numbers to section a sorted array and search through segments with sizes of consecutive fibonacci numbers in O(log n).
